# 软链接（符号链接）使用指南

## 概述

软链接（Symbolic Link）是一种特殊的文件，指向另一个文件或目录。在web2py项目中，可用于跨应用共享模板、CSS、JS等资源。

---

## Linux 系统

### 基本语法

```bash
ln -s <目标文件> <链接名称>
```

### 创建文件链接

```bash
# 创建layout.html链接到service_center的layout
cd /opt/<web2py_service>/applications/<legacy_app>/views
ln -s ../../service_center/views/layout.html layout.html
```

### 创建目录链接

```bash
# 共享静态资源目录
cd /opt/<web2py_service>/applications/<legacy_app>/static
ln -s ../../service_center/static/common common
```

### 查看链接

```bash
ls -la layout.html
# 输出: layout.html -> ../../service_center/views/layout.html
```

### 删除链接

```bash
rm layout.html  # 只删除链接，不影响原文件
```

### 更新链接

```bash
rm layout.html
ln -s <新目标> layout.html
```

## Windows 系统

### 前提条件

1. **管理员权限**：必须以管理员身份运行命令提示符
2. **开发者模式**（Windows 10/11）：设置 → 更新和安全 → 开发者选项 → 开启

### 基本语法

```cmd
mklink <链接名称> <目标文件>           # 文件链接
mklink /D <链接名称> <目标目录>        # 目录链接
```

### 创建文件链接

```cmd
# 以管理员身份运行cmd
cd C:\web2py\applications\<legacy_app>\views
mklink layout.html ..\..\service_center\views\layout.html
```

### 创建目录链接

```cmd
cd C:\web2py\applications\<legacy_app>\static
mklink /D common ..\..\service_center\static\common
```

### 查看链接

```cmd
dir layout.html
# 显示: <SYMLINK> layout.html [...\service_center\views\layout.html]
```

### 删除链接

```cmd
del layout.html        # 删除文件链接
rmdir common           # 删除目录链接（不会删除原目录内容）
```

### PowerShell 方式

```powershell
# 以管理员身份运行PowerShell
New-Item -ItemType SymbolicLink -Path "layout.html" -Target "..\..\service_center\views\layout.html"
```

## web2py 项目应用场景

### 1. 共享Layout模板

```
service_center/views/layout.html     ← 主模板（真实文件）
<legacy_app>/views/layout.html               ← 符号链接
other_app/views/layout.html          ← 符号链接
```

**优点**：修改一处，所有应用同步更新

### 2. 共享CSS/JS资源

```
service_center/static/css/app.css    ← 真实文件
<legacy_app>/static/css/app.css              ← 符号链接
```

### 3. 共享组件目录

```
service_center/static/components/    ← 真实目录
<legacy_app>/static/components/              ← 目录链接
```

## 路径说明

### 相对路径 vs 绝对路径

| 类型 | 示例 | 推荐场景 |
|------|------|----------|
| 相对路径 | `../../service_center/views/layout.html` | 项目内部引用 |
| 绝对路径 | `/opt/<web2py_service>/applications/service_center/views/layout.html` | 跨项目引用 |

**推荐使用相对路径**：项目迁移时无需修改链接

### 路径层级说明

```
<legacy_app>/views/layout.html
     ↓
../../                              # 上两级到applications目录
service_center/views/layout.html    # 目标文件
```

## 常见问题

### 1. 权限不足（Windows）

**错误**：`你没有足够的权限执行此操作`

**解决**：
- 以管理员身份运行cmd
- 或开启Windows开发者模式

### 2. 链接失效

**原因**：目标文件被移动或删除

**检查**：
```bash
# Linux
ls -la layout.html
file layout.html

# Windows
dir layout.html
```

### 3. 相对路径错误

**错误**：`No such file or directory`

**排查**：
```bash
# 从链接所在目录验证相对路径
cd /opt/<web2py_service>/applications/<legacy_app>/views
ls -la ../../service_center/views/layout.html
```

### 4. Git版本控制

**问题**：符号链接在不同系统间可能不兼容

**解决方案**：
```bash
# .gitattributes
*.html text eol=lf

# 或在部署脚本中创建链接
```

## 部署脚本示例

### Linux (deploy.sh)

```bash
#!/bin/bash
APP_DIR="/opt/<web2py_service>/applications"

# 创建<legacy_app>的layout链接
cd "$APP_DIR/<legacy_app>/views"
rm -f layout.html
ln -s ../../service_center/views/layout.html layout.html

# 创建其他应用的链接
cd "$APP_DIR/other_app/views"
rm -f layout.html
ln -s ../../service_center/views/layout.html layout.html

echo "符号链接创建完成"
```

### Windows (deploy.bat)

```batch
@echo off
set APP_DIR=C:\web2py\applications

:: 创建<legacy_app>的layout链接
cd /d "%APP_DIR%\<legacy_app>\views"
if exist layout.html del layout.html
mklink layout.html ..\..\service_center\views\layout.html

:: 创建其他应用的链接
cd /d "%APP_DIR%\other_app\views"
if exist layout.html del layout.html
mklink layout.html ..\..\service_center\views\layout.html

echo 符号链接创建完成
pause
```

## 替代方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **符号链接** | 实时同步、无冗余 | Windows需管理员权限 | Linux服务器 |
| **硬链接** | 无需特殊权限 | 不能跨分区、仅限文件 | 同分区文件共享 |
| **文件复制** | 简单、兼容性好 | 需手动同步 | 小型项目 |
| **URL引用** | 无需链接 | 仅限CSS/JS | 静态资源 |

---

## 最佳实践

1. **使用相对路径**：便于项目迁移
2. **部署脚本化**：确保环境一致性
3. **文档记录**：记录所有链接关系
4. **定期检查**：验证链接有效性
5. **Git处理**：在.gitignore中忽略链接，部署时创建
