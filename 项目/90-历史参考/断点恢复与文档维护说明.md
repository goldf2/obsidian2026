# 断点恢复与文档维护说明

> 生成日期: 2026-06-15  
> 目的: 说明会话断开后如何恢复开发上下文，以及本次生成的系统文档如何继续维护。

## 1. 关于断开的聊天会话

当前环境不能直接读取已经断开的聊天原文。可恢复的是本地工作断点，包括:

- 当前仓库文件。
- `git status` 中的未提交改动。
- 已存在的 README、规范文档、应用内架构说明。
- 代码中的菜单、控制器、模型和权限定义。
- 当前生成的新文档。

因此后续恢复任务时，不应依赖“记住上次聊天”，而应优先查看本文和下面两个总览文档。

## 5. 每次新增功能的维护流程

```mermaid
flowchart LR
    A[确认业务归属] --> B[判断菜单层级]
    B --> C[更新导航或页内菜单]
    C --> D[登记页面路由]
    D --> E[登记权限和数据来源]
    E --> F[补截图索引]
    F --> G[更新清理清单或待确认问题]
```

执行规则:

1. 系统级入口更新顶部导航。
2. 业务域或模块入口更新平台左侧菜单。
3. 当前 app 内部页面更新 `response.page_menu`。
4. 新增、编辑、删除、导入、导出、审核、刷新等动作放页面工具栏，不放平台侧栏。
5. 新路由必须登记 URL、控制器、方法、页面用途、权限和数据来源。
6. 核心页面需要登记桌面和移动截图路径。
