## 1. 问题背景

### 1.1 原有问题

| 问题 | 描述 |
|------|------|
| 代码重复 | `layout.html` 和 `frame.html` 导航栏代码重复约50行 |
| HTML结构错误 | `mgr/index.html` extend了layout却包含完整`<html>`标签 |
| 首页无内容 | 仅嵌入iframe到外部应用 `jsll002` |
| CSS分散 | 样式内联在多个模板中，难以维护 |
| 文件冗余 | 存在多个`.bak`备份文件 |

### 1.2 原有文件结构

```
views/
├── layout.html          # 主布局（含内联CSS）
├── layout.html.bak
├── sidebar.html         # 侧边栏（CSS+HTML+JS混合）
├── sidebar.html.bak
├── default/
│   ├── index.html       # 首页（仅iframe）
│   ├── index.html.bak
│   └── frame.html       # 独立框架（与layout重复）
└── mgr/
    └── index.html       # HTML结构错误
```

## 2. 优化方案

### 2.1 设计原则

1. **DRY原则** - 消除重复代码
2. **关注点分离** - CSS独立、模板专注结构
3. **Bootstrap优先** - 尽量使用内置类，减少自定义CSS
4. **模块化** - 便于维护和组合

### 2.2 文件变更

| 操作 | 文件 | 说明 |
|------|------|------|
| 新建 | `static/css/app.css` | 统一样式文件 |
| 重写 | `views/layout.html` | 整合sidebar和面包屑 |
| 重写 | `views/default/index.html` | 卡片/列表双视图首页 |
| 重写 | `views/mgr/index.html` | 修复HTML结构 |
| 删除 | `views/sidebar.html` | 已合并到layout |
| 删除 | `views/default/frame.html` | 功能已合并到layout |

## 3. 新架构说明

### 3.1 文件结构

```
views/
├── layout.html              # 统一布局模板
├── default/
│   └── index.html           # 首页（卡片/列表视图）
└── mgr/
    └── index.html           # 管理首页

static/css/
└── app.css                  # 自定义样式（仅布局必需）
```

### 3.2 layout.html 功能模块

```
┌─────────────────────────────────────────────────┐
│  顶部导航栏 (.navbar-top)                        │
│  - 品牌标题                                      │
│  - 主菜单 (response.menu)                        │
│  - 用户菜单                                      │
├──────────┬──────────────────────────────────────┤
│          │  面包屑导航 (可选)                    │
│  侧边栏   │  response.breadcrumb 控制显示        │
│          ├──────────────────────────────────────┤
│ response │                                      │
│ .sub_menu│  主内容区 (.main-content)             │
│          │  {{include}} 子模板内容               │
│          │                                      │
└──────────┴──────────────────────────────────────┘
```

### 3.3 控制器配置示例

```python
def index():
    response.title = '页面标题'
    response.menu = rmenu              # 顶部导航
    response.sub_menu = index_sub_menu # 侧边栏菜单
    response.breadcrumb = [            # 面包屑（可选）
        ('系统设置', URL('mgr', 'index')),
        ('用户管理', None)  # 最后一项为当前页
    ]
    return dict()
```

## 4. 自定义CSS说明

`static/css/app.css` 仅包含Bootstrap 4无法实现的必要样式：

### 4.1 布局定位（必须）

```css
/* 固定顶部导航 */
.navbar-top { position: fixed; top: 0; height: 48px; z-index: 1030; }

/* 固定侧边栏 */
.sidebar { position: fixed; top: 48px; left: 0; width: 220px; }

/* 内容区偏移 */
.main-content { margin-left: 220px; margin-top: 48px; }
```

### 4.2 侧边栏菜单（必须）

```css
/* 菜单项样式 */
.sidebar-menu a { padding: 10px 16px; border-left: 3px solid transparent; }
.sidebar-menu a:hover { border-left-color: #007bff; }

/* 子菜单展开 */
.sidebar-menu .submenu { display: none; }
.sidebar-menu .open > .submenu { display: block; }

/* 展开箭头 */
.sidebar-menu .has-sub > a::after { content: '\f105'; font-family: FontAwesome; }
```

### 4.3 响应式适配（必须）

```css
@media (max-width: 768px) {
  .sidebar { transform: translateX(-100%); }  /* 默认隐藏 */
  .sidebar.show { transform: translateX(0); } /* 展开 */
  .main-content { margin-left: 0; }           /* 全宽 */
}
```

### 4.4 首页卡片网格（可选）

```css
.home-cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 16px;
}
```

## 5. 首页功能说明

### 5.1 双视图切换

- **卡片视图**: 功能模块以卡片形式展示
- **列表视图**: 功能模块以列表形式展示
- 用户偏好自动保存到 `localStorage`

### 5.2 动态生成

首页卡片/列表根据 `response.sub_menu` 自动生成，无需手动维护。

```python
# 在 models/xx_menu.py 中定义
index_sub_menu = [
    ('生产活动', False, '#', [
        ('作业指导书', False, URL('cx1', 'zyzd', 'index')),
        ('工装看板', False, URL('cx1', 'gzkb', 'gz_board')),
    ]),
    ('创新模具西安分公司', False, URL('cx1', 'xian', 'index')),
]
```

## 6. 使用Bootstrap类清单

以下Bootstrap 4类在模板中广泛使用：

| 类别 | 使用的类 |
|------|---------|
| 布局 | `container-fluid`, `d-flex`, `justify-content-between`, `align-items-center` |
| 间距 | `mb-0`, `mb-2`, `mb-3`, `mb-4`, `mt-4`, `py-2`, `py-5`, `px-3`, `p-0` |
| 颜色 | `bg-white`, `bg-light`, `bg-transparent`, `text-dark`, `text-muted` |
| 边框 | `border-right`, `border-bottom`, `rounded-circle` |
| 组件 | `card`, `card-body`, `card-header`, `badge`, `btn`, `btn-group`, `btn-sm` |
| 导航 | `navbar`, `nav-item`, `nav-link`, `dropdown`, `breadcrumb` |
| 效果 | `shadow`, `shadow-sm`, `fade`, `show` |
| 列表 | `list-unstyled` |
| 文本 | `text-center`, `text-left`, `text-decoration-none`, `font-weight-normal`, `small`, `display-4` |

## 7. 后续扩展建议

### 7.1 添加新页面

1. 创建 `views/controller/action.html`
2. 首行添加 `{{extend 'layout.html'}}`
3. 控制器设置 `response.sub_menu` 启用侧边栏

### 7.2 添加面包屑

```python
response.breadcrumb = [
    ('父级页面', URL('controller', 'parent')),
    ('当前页面', None)
]
```

### 7.3 自定义侧边栏菜单

```python
# 为特定页面使用不同菜单
response.sub_menu = custom_menu_list
```
