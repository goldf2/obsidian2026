# <web2py_service> 项目文档模板

本文档用于每个 web2py 服务项目的项目级说明。通用规范放在仓库根目录各规范分类中，项目目录只记录该项目自己的事实、差异和执行状态。

## 必备文档

每个项目文件夹至少包含：

```text
projects/<web2py_service>/
├── README.md
├── 应用清单.md
├── 运行数据与配置.md
└── 规范对照.md
```

## README.md

建议包含：

- 项目名称
- 部署模式: 云服务商 PaaS / Coolify / Docker Compose / 传统 systemd
- 代码目录或仓库地址
- 运行数据目录或持久化 volume
- systemd 服务、PaaS 应用名或 Coolify 应用名
- 主要入口域名或端口
- 项目定位
- 关联通用规范入口

## 应用清单.md

建议字段：

| App | 类型 | 状态 | 说明 |
| --- | --- | --- | --- |
| `<platform_app>` | 平台 app | 当前 | 统一入口、认证、菜单 |
| `<business_app>` | 业务 app | 当前 | 业务域说明 |
| `<legacy_app>` | 存量 app | 待治理 | 历史功能 |

## 运行数据与配置.md

建议字段：

| 项目 | 路径/值 | 状态 | 说明 |
| --- | --- | --- | --- |
| 部署模式 | 云服务商 PaaS / Coolify / Docker Compose / 传统 systemd | 当前/待迁移 | 当前生产运行方式 |
| 代码目录/仓库 | `/opt/<web2py_service>` 或 Git 仓库地址 | 当前 | 传统部署记录实例路径，容器化部署记录仓库 |
| 数据目录/持久化存储 | `/data/<web2py_service>`、PaaS 持久化磁盘或对象存储 | 当前/待建 | 外置运行数据，PaaS/容器化部署必须持久化 |
| 运行服务 | `<web2py-service>.service`、PaaS 应用名或 Coolify 应用名 | 当前 | 记录实际运行入口 |
| 配置 | `/data/<web2py_service>/<app>/config/appconfig.ini` | 当前/待治理 | 生产配置 |

## 规范对照.md

建议按通用规范逐项记录执行状态：

| 规范项 | 状态 | 说明 |
| --- | --- | --- |
| 代码和运行数据分离 | 待检查 | - |
| 统一认证 | 待检查 | - |
| 统一导航 | 待检查 | - |
| 上传目录外置 | 待检查 | - |
| 备份策略 | 待检查 | - |
| CI/CD | 待检查 | - |
| Docker/Compose 基线 | 待检查 | - |
| PaaS 部署状态 | 待检查 | - |
| 文档归档 | 待检查 | - |
