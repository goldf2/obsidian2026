S0 现状保护与盘点规范

1. 阶段定位

S0 是正式治理前的保护阶段，只做原始备份、现状盘点和忽略规则草案，不做正式 Git 初始提交。

2. 阶段目标

  • 保护现有代码、数据库、上传文件、真实配置和 runtime 现场。
  • 识别哪些内容属于代码，哪些属于运行数据。
  • 为 S1-S4 提供可信的现状基线。

3. 必做事项

  • 备份原始目录。
  • 盘点项目、app、代码目录、数据库、uploads、config、runtime。
  • 输出 .gitignore 草案。
  • 标记敏感文件、密钥文件和不得入库目录。

4. 交付物

  • 原始备份清单。
  • 代码与运行数据盘点表。
  • .gitignore 草案。
  • 敏感文件识别清单。

5. 验收标准

  • 原始目录可恢复。
  • 已识别运行数据与真实配置位置。
  • 已明确正式建仓时需要排除的文件和目录。
