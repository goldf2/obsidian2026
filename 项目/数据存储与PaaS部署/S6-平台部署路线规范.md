# S6 平台部署路线规范

## 1. 阶段定位

S6 负责把 Docker 化后的交付物适配到不同平台路线，包括云服务商 PaaS、Coolify 和自建 Compose。

## 2. 阶段目标

- 保持 web 开发规范平台中立。
- 将平台专用配置与应用代码隔离。
- 分别定义 S6A、S6B、S6C 的适用边界。

## 3. 阶段分支

- `S6A 云服务商 PaaS`: 托管平台部署、日志、备份、监控和域名。
- `S6B Coolify`: 自托管 PaaS 路线。
- `S6C 自建 Compose`: 过渡、内网或低成本路线。

## 4. 必做事项

- 明确平台选择标准。
- 明确平台专用配置边界。
- 明确不同平台的数据挂载和日志方式。

## 5. 交付物

- 平台选型说明。
- 平台配置模板。
- 部署与回滚入口说明。

## 6. 验收标准

- 平台配置不污染业务代码。
- 同一代码基线可切换不同平台路线。
