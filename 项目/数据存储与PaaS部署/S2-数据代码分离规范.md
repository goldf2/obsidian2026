# S2 数据代码分离规范

## 1. 阶段定位

S2 明确代码与运行数据边界，确保 Git 只管理代码、模板和文档，不管理真实运行数据。

## 2. 阶段目标

- 明确哪些文件未来可以进入 Git。
- 明确哪些数据库、上传文件、真实配置和 runtime 必须排除。
- 为存储外置和 Docker 化建立边界。

## 3. 必做事项

- 梳理代码目录与数据目录边界。
- 标记数据库、uploads、config、sessions、errors、cache、logs。
- 输出发布不覆盖数据的规则。

## 4. 交付物

- 数据与代码边界表。
- 不入 Git 目录清单。
- 发布前敏感文件检查规则。

## 5. 验收标准

- 代码发布不覆盖生产数据。
- Git 不纳入数据库、上传文件和真实配置。
