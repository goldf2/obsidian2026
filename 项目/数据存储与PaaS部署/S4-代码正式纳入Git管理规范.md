# S4 代码正式纳入 Git 管理规范

## 1. 阶段定位

S4 在 S1-S3 清理完成后进行，目标是形成清理后的代码基线，让后续治理可追溯、可审查、可回滚。

## 2. 阶段目标

- 初始化正式代码仓库。
- 固化 `.gitignore` 和初始代码基线。
- 建立分支、提交和基础检查规则。

## 3. 必做事项

- 初始化仓库。
- 补齐 `.gitignore`。
- 提交清理后的初始代码基线。
- 加入基础敏感文件和目录检查。

## 4. 交付物

- 仓库初始化清单。
- `.gitignore` 模板。
- 初始提交说明。
- 分支策略和提交规范。

## 5. 验收标准

- 仓库中不存在数据库、uploads、真实配置和 runtime。
- 初始提交能还原清理后的代码目录。
- 后续改造可通过 diff 审查。
