# S3 存储外部独立化规范

## 1. 阶段定位

S3 把数据库、上传文件、真实配置和 runtime 从 app 目录中独立出来，使代码目录可重建、数据不丢失。

## 2. 阶段目标

- 建立统一的外部数据路径规则。
- 使 app 目录只承担代码和模板职责。
- 为 Git 纳管和 Docker 化做好持久化边界准备。

## 3. 必做事项

- 设计 `/data/<web2py_service>/<app>` 逻辑路径。
- 外置数据库、uploads、config、runtime。
- 明确权限、迁移步骤和回滚方式。

## 4. 交付物

- 外部数据目录规范。
- 数据迁移步骤模板。
- 验证与回滚清单。

## 5. 验收标准

- 重建 app 代码目录后数据不丢。
- 上传、下载、登录、日志和错误票据仍可用。
